# 集GO（しゅうごー）

LINE などで招待リンクを送り、**相手が承認したときだけ**、お互いの現在地をリアルタイムで地図に表示する待ち合わせアプリ（PWA）。**1グループ最大5人**まで同時共有できます。「みんなで集まってGO」＝**集GO**。

自販機マップと同じ構成（vanilla JS + Leaflet + Firebase + PWA、ビルド不要）。

## できること（MVP / v1）
- 表示名を設定 → 「待ち合わせを作成」で招待リンクを発行
- 「LINEで送る / 共有」から共有シート経由でLINEに送付（未対応端末はコピー）。**同じリンクを複数人に送れます（最大5人）**
- 相手がリンクを開く → 「○○さんが現在地を共有したいです」→ **承認**（満員時は参加不可）
- 参加メンバー全員の現在地を Leaflet 地図にリアルタイム表示（**人別の色分けピン**・自分からの距離・各メンバーへのライン・測位精度の円）
- **メンバー一覧**（色・名前・距離、タップでフォーカス＆方向表示）
- **集合場所ピン**：地図タップ／**住所・場所名で検索**（Nominatim）して設定、両者で共有、各自からの距離＋**徒歩ETA**を表示
- **合流判定**：メンバーごとに約50mで「○○さんがまもなく合流」、約20mで「○○さんと合流🎉」＋バイブレーション通知。バナーには最寄りの人を表示
- **方向コンパス**：フォーカス中のメンバーがどの方角かを矢印で表示（端末コンパス有効時は実際に向くべき向きを指す）
- **状況の共有**：「向かっています」「5分で着きます」等のプリセットを送信、各メンバーのバナーに表示（複数同時）
- **ホーム画面に追加（A2HS）**：招待を開いたとき「ホーム画面に追加しますか？」を表示。はい→各OSに応じて案内（下記）

## ホーム画面への追加について（重要な制約）
OSの仕様上、**タップ無しの完全自動追加はどのプラットフォームでも不可**（勝手なインストールは禁止）。本アプリは「できる範囲で最短」を実装：
- **Android（Chrome/Edge）**：`beforeinstallprompt` を利用し、はい→OS標準のインストールダイアログ。確認1回でアイコン設置。
- **iPhone（Safari）**：プログラムからの追加APIが存在しないため、はい→「共有→ホーム画面に追加」の手順を図解で案内。
- **LINEのアプリ内ブラウザ**：インストール不可。「ブラウザ（Safari/Chrome）で開く」を案内 → そこで追加。
- すでに追加済み（standalone起動）や「あとで」を押した場合は再表示しません（localStorage 記憶）。
- 実際に機能するのは **HTTPS 配信時**（GitHub Pages 等）。

## 悪用対策（設計）
- **相互同意**：相手が承認するまで位置は共有されない
- **時間制限**：2時間で自動失効（`SESSION_TTL_MS`）。残り時間を常時表示
- **即終了**：どちらでもワンタップで共有停止
- **常時明示**：共有中は「○○さんと共有中」バナーを表示（こっそり追跡不可）
- **前面時のみ更新**：アプリを離れると位置更新を一時停止
- **履歴を残さない**：位置は「現在地」を上書きするだけ（履歴を保存しない）。退出時 `onDisconnect` で自分の位置を自動削除
- **なりすまし防止**：匿名認証 + ルールで「自分の位置ノードしか書けない」
- **推測不能なID**：セッションIDは `crypto` 製のランダム値

## セットアップ
1. [FIREBASE_SETUP.md](FIREBASE_SETUP.md) に沿って Firebase（Realtime Database + 匿名認証）を用意し、`index.html` に config を貼る
2. ローカル：`pwsh ./serve.ps1` → http://localhost:8080/
3. 2台 / 別端末のテストは **HTTPS 必須**（位置情報の制約）。GitHub Pages 等にデプロイして確認
